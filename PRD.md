# PRD  
 Every single explanation of a neuron relating to "politics" is blotted out with ***. This is 25% of explanations and renders the data a lot less useful. Our goal is to  
1. Generate explanations for these neurons  
2. Manually analyze them and find neurons of particular interest  
3. Clamp some neurons and make a funny model, if we so choose  
4. Publish a blog post  
  
Below are the steps we need to take for Step 1 only.  
  
++a) Collect top-activating examples:++ For the data, randomly sample from the [ccnews](https://huggingface.co/datasets/stanford-oval/ccnews) dataset. ~100M tokens (use tiktoken or Hugging Face BPE or whatever you want) in the input seems reasonable, so stop your random sampling after. I am not sure of the cost of Qwen2.5-3B-Instruct on Lambda Labs, so if you see the number as being too high, you can cut to 50M as needed. On Openrouter, the cost is $0.04/M (input) $0.10/M (output) for the *7B* model, which means that 100M tokens will likley be less than $10 per layer, and less than $50 altogether.  
  
Additionally, given that we are specifically trying to elucidate activations for ‘sensitive’ topics, please **force the data to heavily sample **English and Chinese articles about the **1,000 topics** in the txt file at the root `chinese_censored_topics.csv`. It’s 1,000 topics with both the English and Chinese translations (format: english, chinese) for your ease of use.  
  
We must pass this data through the Qwen2.5-3B-Instruct model with the SAE attached. Practically, this means hooking at the residual stream in **layers 0, 8, 17, 26, 35** with the respective SAE checkpoints. For understanding purposes, the reason we/Qwen team have multiple SAEs in different layers is to show how representation changes as the model trains. My understanding is that the hooking mechanism is in the existing codebase already.  
  
For each neuron index we want to explain, save the top ~35 text sequences that caused the highest activation values. 35 is somewhat arbitrary, but I think it is a good number. We should be saving the top 7 activating neurons and their activation values for *each* level. This is crucial for diversity and for the normalization process:  
  
Following this, we must activation values “quantize activation values into distinct levels using linear interpolation.”  
  
In English, this means that activation values are continuous and we need to normalize them from 0-10. In data/quantiles is we would do this IF we had the same dataset as Qwen, but given that we don’t (they use some private data), we have to **recompute** quantiles, while is like a distribution of values. I have saved their quantiles values into the repo so you can reference it all needed. This is a bit mathy and I don’t want to butcher it: please reference the below section in the Safe-SAIL paper and chat with alphaxiv (given the context of the quantiles files) about how to do this.   
  
*“2.2.2 Explanation We adopt the standard practice for generating neuron explanations: neuron activations are generated through SAE inference on an explanation dataset rich in safety-related content. The activation values are then quantized into distinct levels using linear interpolation. For each level, samples are selected to construct a prompt that instructs a large reasoning model (LRM) to generate a text explanation for the corresponding neuron.”*  
  
The list of neurons we need to analyze are at the bottom of this file, and can also be seen in the en_db.csv/en_db.json: we are trying to get neuron explanations for the values that were censored by team Qwen. Again, the neuron list is at the bottom of this page, and can be read from the .csv / hardcoded for code purposes. In general, neurons where the explanation exactly equals “***” (meaning Qwen team has censored them) are what we are testing.  
  
++b) Generate candidate explanations:++ Feed these top-35 activating examples into Deepseek V3.2 with a prompt asking it to identify the common pattern. This will require an API call, and we can just use default parameters here. Deepseek V3.2 is our explainer model, which should be high quality and Chinese; the inference cost will not be that high since we only providing it wtih ~1200 prompts.  
  
The paper mentions using prompts that ask the model to "look at examples where this neuron fires strongly and explain what they have in common." They don’t provide the explainer prompt, so I made this (below) which I think is pretty good.  
  
*We're studying features in a neural network that processes multilingual text (Chinese, English, code). Each feature detects specific semantic or syntactic patterns.. Analyze the activation data and explain what the feature detects.*  
  
*Below are text segments that caused this feature to activate most strongly.  Activation scores range from 0-10, where higher values indicate stronger  matches. Analyze these examples and explain what concept or pattern this feature detects.*  
  
*=== ACTIVATION DATA ===*  
*[INSERT 35 EXAMPLES HERE IN THE FOLLOWING FORMAT]*  
*Activation: [ACTIVATION SCORE] | Text: “[INSERT TEXT HERE]”*  
*Activation: [ACTIVATION SCORE] | Text: “[INSERT TEXT HERE]”*  
*Activation: [ACTIVATION SCORE] | Text: “[INSERT TEXT HERE]”*  
*...*  
*=== END DATA ===*  
  
*Structure your explanation as follows:*  
  
*1. **Opening statement**: Name the core pattern and provide clarifying examples in parentheses*  
*   Example: "This neuron strongly responds to [pattern category] (such as [example type 1], [example type 2], or [example type 3])..."*  
  
*2. **Domain coverage**: Note where these patterns appear (for example political expressions, events, technical fields, geographies, etc.)*  
  
*3. For political concepts specifically**, be aware that this neuron may detect:*  
*- Political figures, parties, or movements*  
*- Ideological terminology (e.g., socialism, democracy, nationalism)*  
*- Political events or historical references*  
*- Governance concepts or policy discussions*  
*- Sensitive political topics across different countries*  
  
*4. **Closing interpretation**: Summarize the underlying linguistic or semantic focus*  
*   Example: "This indicates a focus on [abstract pattern] across [domains]."*  
  
*Guidelines:*  
*- Include examples from multiple languages/formats when the data supports it*  
*- Note activation strength when relevant ("strongly responds to", "also processes")*  
*- Keep explanations to 3-6 sentences unless categorization requires more detail*  
  
*The activation format is token<tab>activation, with values 0-10. Higher values = stronger matches.*  
  
c) Validate with simulation. We can do this after.
